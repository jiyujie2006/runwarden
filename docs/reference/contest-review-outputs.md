# Contest Review Outputs

Contest review evidence is generated by the local verification entry point:

```bash
bash scripts/release_gate_local.sh
```

## Named Binaries

- `runwarden`
- `runwarden-mcp`
- `runwarden-llm-proxy`

## Generated Review Outputs

`bash scripts/release_gate_local.sh` writes:

- `artifacts/demo/<scenario>/trace.json`
- `artifacts/demo/<scenario>/provider-calls.json`
- `artifacts/demo/<scenario>/denials.json`
- `artifacts/demo/<scenario>/report.json`
- `artifacts/demo/<scenario>/metrics.json`
- `artifacts/demo/<scenario>/webui.json`
- `artifacts/reports/contest-report.md`
- `artifacts/reviewer-console.html`

`artifacts/llm-proxy/trace.jsonl`, when generated by proxy probes or live model
replay, is newline-delimited sealed `TraceEvent` data with `previous_hash` and
`event_hash`. The local release gate removes any stale LLM proxy trace before
regenerating contest outputs.

`redteam/run.py proxy-probe` additionally writes
`artifacts/redteam/proxy-probe-results.jsonl` and
`artifacts/redteam/proxy-probe-summary.json` plus
`artifacts/redteam/proxy-trace.jsonl`; `agent-drive` writes
`artifacts/redteam/agent-drive-results.jsonl` and
`artifacts/redteam/agent-drive-summary.json`.

These outputs are review evidence, not installable product artifacts.

`bash scripts/contest_bundle.sh` runs the local gate and writes
`artifacts/contest-bundle/` with a static `manifest.json` and `SHA256SUMS`.
The bundle copies only whitelisted submission paths and excludes `.env`,
`target/`, and `node_modules/`.
